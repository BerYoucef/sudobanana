SQL Injection

Category: Web Security | Risk Level: High | Last Updated: June 2025
Home SQL Injection Discussion Edit
SQL Injection is a code injection technique that exploits security vulnerabilities in an application's software when user input is not properly sanitized. It's one of the most common web application vulnerabilities and can lead to unauthorized access to sensitive data.

How SQL Injection Works

SQL injection occurs when malicious SQL statements are inserted into application entry points, such as web form input fields. When the application processes this input without proper validation, the malicious SQL code gets executed by the database.
# Vulnerable code example:
query = "SELECT * FROM users WHERE username = '" + username + "' AND password = '" + password + "'"

# Malicious input:
username = "admin'--"
password = "anything"

# Resulting query:
"SELECT * FROM users WHERE username = 'admin'--' AND password = 'anything'"

Types of SQL Injection

There are several types of SQL injection attacks:
1. In-band SQL Injection: The attacker uses the same communication channel to launch the attack and gather results. This includes Error-based and Union-based injections.
2. Inferential SQL Injection: Also known as "Blind SQL Injection," where the attacker sends data payloads and observes the web application's response and behavior.
3. Out-of-band SQL Injection: The attacker uses different communication channels for the attack and gathering results.
Pro Tip: Always use parameterized queries or prepared statements. It's like using a proper technique - it prevents the messy situations!

Prevention Techniques

The most effective way to prevent SQL injection is to use parameterized queries (prepared statements) and input validation:
# Safe code using parameterized queries:
cursor.execute("SELECT * FROM users WHERE username = %s AND password = %s", (username, password))

# Input validation example:
if re.match("^[a-zA-Z0-9_]+$", username):
    # Process the input
else:
    # Reject invalid input

Impact and Consequences

Successful SQL injection attacks can result in:
